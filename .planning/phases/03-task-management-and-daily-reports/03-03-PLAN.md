  <action>
1. Install date-fns in the web workspace:
   `pnpm add date-fns --filter @daily-report/web`

2. Create `apps/web/src/app/(dashboard)/reports/page.tsx`:
   - 'use client' directive
   - On mount, redirect to `/reports/${format(new Date(), 'yyyy-MM-dd')}` using Next.js `useRouter().replace()`
   - Show a simple loading state while redirecting

3. Create `apps/web/src/app/(dashboard)/reports/[date]/page.tsx`:
   - 'use client' directive
   - Extract `date` from `useParams()`
   - Validate date format (YYYY-MM-DD regex), redirect to today if invalid

   **Team resolution:**
   - Fetch user's teams via GET /teams/my (existing endpoint)
   - Auto-select first team (v1 users have one team)
   - Store teamId in state
   - If no teams, show message "Join a team to start logging tasks" with link to /teams

   **Data fetching:**
   - useQuery with key ['reports', 'daily', date, teamId] to fetch GET /reports/daily?date={date}&teamId={teamId}
   - Enabled only when teamId is available

   **Date navigation:**
   - Previous/Next day buttons using date-fns addDays/subDays
   - Display formatted date using date-fns format (e.g., "Thursday, March 6, 2026")
   - "Today" button to jump to current date
   - Navigation updates the URL via router.push(`/reports/${newDate}`)
   - Highlight if viewing today vs a past/future date

   **Task list:**
   - Display tasks in a list/card layout
   - Each task shows: title, estimated hours, source link (clickable if present), notes
   - Show total hours at the bottom (from API response totalHours)
   - If report is draft: show edit and delete buttons on each task
   - If report is submitted: show "Submitted" badge, hide edit/delete buttons, show submitted_at timestamp

   **Task creation form:**
   - Only visible when report status is 'draft' (or no report exists yet)
   - Use react-hook-form with zod resolver (createTaskSchema from shared)
   - Fields: title (required), estimatedHours (required, number input step 0.25), sourceLink (optional, url), notes (optional, textarea)
   - Hidden fields: reportDate (from URL param), teamId (from resolved team)
   - useMutation for POST /tasks, on success invalidate ['reports', 'daily', date, teamId] query and reset form
   - Show toast on success/error using sonner

   **Task editing:**
   - Click edit button -> toggle task row to inline edit mode (or open a modal/inline form)
   - Pre-populate react-hook-form with current values
   - useMutation for PATCH /tasks/:id, on success invalidate query
   - Cancel button to exit edit mode

   **Task deletion:**
   - Click delete button -> confirm with window.confirm() or inline confirmation
   - useMutation for DELETE /tasks/:id, on success invalidate query
   - Show toast on success

   **Report submission:**
   - "Submit Report" button visible only when: report exists, status is 'draft', and tasks.length > 0
   - Click -> confirm dialog ("Once submitted, you cannot edit this report")
   - useMutation for POST /reports/:id/submit
   - On success: invalidate query, show success toast
   - After submission: UI updates to show submitted state (edit/delete hidden, submit button hidden)

   **Styling:**
   - Use Tailwind CSS classes consistent with existing pages (teams, settings)
   - Card-based layout for the report area
   - Responsive: single column on mobile, comfortable spacing on desktop
   - Use existing UI patterns from teams/settings pages for consistency (Card component if one exists, or div with shadow/rounded)
