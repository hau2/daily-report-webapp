3. Create `apps/api/src/tasks/tasks.service.ts` injecting SupabaseService:
   - `getOrCreateReport(userId, teamId, date)`: Query daily_reports for existing. If found, return it. If not, insert with status='draft' and return. Use `.upsert()` with `onConflict: 'user_id,team_id,report_date'` to handle race conditions per RESEARCH.md Pitfall 2.
   - `assertReportEditable(reportId)`: Fetch report status, throw ForbiddenException if 'submitted'.
   - `assertReportOwner(reportId, userId)`: Fetch report user_id, throw ForbiddenException if mismatch. Combine with assertReportEditable to avoid extra query when both checks needed.
   - `createTask(userId, dto: CreateTaskDto)`: Call getOrCreateReport(userId, dto.teamId, dto.reportDate). Call assertReportEditable. Insert into tasks table with report_id, title, estimated_hours, source_link, notes. Map snake_case DB columns to camelCase response. Return created task.
   - `updateTask(userId, taskId, dto: UpdateTaskDto)`: Fetch task by id, get report_id. Call assertReportOwner + assertReportEditable. Update task fields (only provided fields). Map response to camelCase.
   - `deleteTask(userId, taskId)`: Fetch task by id, get report_id. Call assertReportOwner + assertReportEditable. Delete task. Return void.
   - `getDailyReport(userId, teamId, date)`: Query daily_reports for user+team+date. If no report, return null. Fetch tasks by report_id ordered by sort_order, created_at. Compute totalHours as sum of estimated_hours. Map to DailyReportWithTasks. Map all snake_case to camelCase.
   - `submitReport(userId, reportId)`: Call assertReportOwner. Check status is 'draft' (throw ForbiddenException if already submitted). Count tasks for report -- if 0, throw BadRequestException('Cannot submit empty report'). Update status to 'submitted', submitted_at to now(). Return updated report.

   DB column mapping: Use a private helper `mapReport(row)` and `mapTask(row)` to convert snake_case DB rows to camelCase TypeScript interfaces. Pattern: report_id -> reportId, user_id -> userId, etc.
