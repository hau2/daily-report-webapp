  <action>
1. Create `apps/api/src/auth/guards/email-verified.guard.ts`:
   - Implement as a NestJS CanActivate guard
   - Inject SupabaseService and Reflector
   - In canActivate(): get the request user (from AccessTokenGuard — it runs first)
   - If no user on request (unauthenticated routes like /auth/register, /auth/login), return true (skip check)
   - Query `users` table for `email_verified` where `id = user.userId`
   - If email_verified is true, return true
   - If email_verified is false, throw ForbiddenException('Email not verified')
   - Add a custom decorator `@SkipEmailVerification()` using SetMetadata('skipEmailVerification', true)
   - In canActivate(), check for this metadata via Reflector — if present, return true (skip check)
   - Export both the guard and the decorator from the file

2. Add `resendVerification` method to `apps/api/src/auth/auth.service.ts`:
   - Accept userId (string) parameter
   - Query the users table for the user by id to get email and email_verified
   - If already verified, throw BadRequestException('Email is already verified')
   - Check for rate limiting: query the user's `verification_email_sent_at` column... Actually, simpler approach: store a `last_verification_sent_at` timestamp in a class-level Map<string, number> (in-memory, sufficient for single-server). If last sent < 60 seconds ago, throw new BadRequestException('Please wait before requesting another verification email')
   - Generate a new JWT verification token (same as in register(): purpose 'email-verification', expiresIn '1h')
   - Call emailService.sendVerificationEmail(user.email, token)
   - Update the in-memory rate limit map
   - Return { message: 'Verification email sent' }

3. Add resend-verification endpoint to `apps/api/src/auth/auth.controller.ts`:
   - `@Post('resend-verification')` with `@UseGuards(AccessTokenGuard)` and `@HttpCode(HttpStatus.OK)`
   - Decorate with `@SkipEmailVerification()` (imported from the new guard file) so unverified users can call it
   - Extract userId from req.user, call authService.resendVerification(userId)

4. Update `apps/api/src/auth/auth.module.ts`:
   - Add EmailVerifiedGuard to providers array
   - Export EmailVerifiedGuard so it can be used as APP_GUARD

   - This makes it run on ALL routes after AccessTokenGuard
   - Import EmailVerifiedGuard from './auth/guards/email-verified.guard'

2. Add `@SkipEmailVerification()` to auth controller routes that unverified users need:
   - In `apps/api/src/auth/auth.controller.ts`, add `@SkipEmailVerification()` decorator to:
     - `getMe()` — the /auth/me endpoint (frontend needs this to check emailVerified status)
   - The other auth routes (register, login, verify-email, forgot-password, reset-password, extension-login, extension-refresh) do NOT use AccessTokenGuard, so the EmailVerifiedGuard should skip them automatically (no user on request). But refresh and logout DO use guards — add @SkipEmailVerification() to refresh() and logout() as well.

3. Also add `@SkipEmailVerification()` to the UsersModule's `GET /users/me` endpoint if it exists (check the file). The user needs to be able to read their profile to see their email on the verify-required page.

4. Update the `acceptInvitation` method in `apps/api/src/teams/teams.controller.ts`:

   - But make the error message specific: add a check at the START of `TeamsService.acceptInvitation()` that queries the user's email_verified status and throws ForbiddenException('You must verify your email before accepting team invitations') if not verified
   - Actually, since the global guard already returns 403 "Email not verified", this is already handled. No change needed to teams controller unless we want a more specific message. Keep it simple — the global guard is sufficient.

5. Update the `GET /auth/me` endpoint to return emailVerified status:

   - The AccessTokenStrategy.validate() only returns userId and email from the JWT

     - Simplest approach: add a `getProfile` method to AuthService that takes userId, queries users table, returns { userId, email, displayName, emailVerified: user.email_verified }
     - Update getMe() to call this method instead of directly returning req.user
