1. Update `apps/web/src/lib/api-client.ts` to handle 403 "Email not verified":
   - In the apiClient function, after the 401 refresh attempt block and before the generic error handling:
   - Add a check: if response.status === 403, try to parse the body. If message is "Email not verified" (or contains "email" and "verified", case-insensitive), redirect to /verify-required:
     ```
     if (response.status === 403) {
       try {
         const body = await response.clone().json();
         if (typeof body?.message === 'string' && body.message.toLowerCase().includes('email not verified')) {
           if (typeof window !== 'undefined') {
             window.location.href = '/verify-required';
           }
           throw new Error(body.message);
         }
       } catch (e) {
         if (e instanceof Error && e.message.toLowerCase().includes('email not verified')) throw e;
       }
     }
     ```
   - This ensures that any 403 "Email not verified" from the global guard redirects the user, even if they somehow bypass the frontend checks
