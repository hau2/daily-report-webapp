  <behavior>
    - removeMember(teamId, ownerUserId, targetUserId): sets left_at on team_members row, throws if target is owner or not a member
    - leaveTeam(teamId, userId): sets left_at on team_members row, throws if user is the owner (owner must transfer first)
    - transferOwnership(teamId, currentOwnerId, targetUserId): updates target role to 'owner', current owner role to 'member', throws if target not a member
    - cancelInvitation(teamId, ownerUserId, inviteeEmail): deletes pending invitation row (where used_at is null), throws if not found
    - deleteTeam(teamId, ownerUserId): deletes all team_members, team_invitations, daily_reports (and their tasks), and the team row itself
    - getTeamReports (manager service): also includes users who have left_at set but have daily_reports for the requested date
  </behavior>
  <action>
    1. Update `TeamManagerGuard` to also reject users who have `left_at` set (departed members should not pass the owner check). Add `.is('left_at', null)` to the query.

    2. Add 5 new methods to `TeamsService`:

    **removeMember(teamId, ownerUserId, targetUserId)**:
    - Query team_members for targetUserId in teamId where left_at is null
    - Throw NotFoundException if not found
    - Throw BadRequestException if target's role is 'owner' (cannot remove owner)
    - Throw BadRequestException if targetUserId === ownerUserId (use leaveTeam instead)
    - Update the row: set `left_at = new Date().toISOString()`

    **leaveTeam(teamId, userId)**:
    - Query team_members for userId in teamId where left_at is null
    - Throw NotFoundException if not found
    - Throw BadRequestException if role is 'owner' ("Owner must transfer ownership before leaving")
    - Update the row: set `left_at = new Date().toISOString()`

    **transferOwnership(teamId, currentOwnerId, targetUserId)**:
    - Query team_members for targetUserId in teamId where left_at is null
    - Throw NotFoundException if target not found
    - Throw BadRequestException if targetUserId === currentOwnerId
    - Update target's role to 'owner'
    - Update current owner's role to 'member'

    **cancelInvitation(teamId, inviteeEmail)**:
    - Delete from team_invitations where team_id = teamId, invitee_email = inviteeEmail, used_at is null
    - If no rows affected (count === 0), throw NotFoundException

    **deleteTeam(teamId)**:
    - First get all daily_report IDs for this team: select id from daily_reports where team_id = teamId
    - Delete tasks for those reports: delete from tasks where report_id in (reportIds)
    - Delete daily_reports: delete from daily_reports where team_id = teamId
    - Delete team_invitations: delete from team_invitations where team_id = teamId
    - Delete team_members: delete from team_members where team_id = teamId
    - Delete team: delete from teams where id = teamId

    3. Add 5 new endpoints to `TeamsController`:

    - `DELETE :id/members/:userId` — @UseGuards(TeamManagerGuard), calls removeMember
    - `POST :id/leave` — no TeamManagerGuard (any active member), calls leaveTeam
    - `POST :id/transfer-ownership` — @UseGuards(TeamManagerGuard), body: TransferOwnershipDto, calls transferOwnership
    - `DELETE :id/invitations/:email` — @UseGuards(TeamManagerGuard), calls cancelInvitation (URL-decode email param)
    - `DELETE :id` — @UseGuards(TeamManagerGuard), calls deleteTeam

    Route ordering in controller: Keep literal routes (like 'invitations/accept') before parameterized routes. Place the new `:id/leave`, `:id/transfer-ownership`, `:id/members/:userId`, `:id/invitations/:email` after existing `:id/members` and `:id/invitations` routes. Place `DELETE :id` last.

    4. Update `getMyTeams` in TeamsService to filter out departed members:
    - Add `.is('left_at', null)` to the team_members query in getMyTeams
    - Add `.is('left_at', null)` to the team_members query in getTeamMembers

    5. Update `ManagerService.getTeamReports` for MEMB-06:
    - Current query: gets team_members for teamId (active members only)
    - Change to: get active members (left_at is null) PLUS departed members who have daily_reports for the given date
    - After getting active members, query daily_reports for teamId + date where user_id NOT IN active member IDs
    - For any departed user_ids found, include them in the member list (mark them as departed somehow -- add a boolean `isDeparted` field to TeamMemberReport or similar, but keep it simple -- just include their data)
    - Update TeamMemberReport in shared types to add optional `departed?: boolean` field

    6. Add unit tests to `teams.service.spec.ts`:
    - removeMember: throws if target is owner, throws if not found, sets left_at on success
    - leaveTeam: throws if user is owner, sets left_at on success
    - transferOwnership: updates roles correctly, throws if target not found
    - cancelInvitation: deletes invitation, throws if not found
    - deleteTeam: deletes all related data in correct order

    7. Also update `getTeamMembers` to only return active members (left_at is null). This is used by the team detail page.

    8. Add a `getPendingInvitations(teamId)` method to TeamsService:
    - Query team_invitations where team_id = teamId AND used_at IS NULL AND expires_at > now
    - Return array of { email: string, invitedAt: string, expiresAt: string }
    - Add GET `:id/invitations` endpoint (before the POST `:id/invitations` route) with @UseGuards(TeamManagerGuard)
    - This is needed for MEMB-04 (cancel invitation) -- the frontend needs to list pending invitations to cancel them
