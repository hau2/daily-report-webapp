  <action>
Replace hardcoded Tailwind color classes across all files using this mapping:

**Background colors:**
- `bg-white` -> `bg-background`
- `bg-gray-50` -> `bg-muted`

**Text colors:**
- `text-gray-500`, `text-gray-600` -> `text-muted-foreground`
- `text-gray-900` -> `text-foreground`
- `text-gray-700` -> `text-foreground`
- `text-gray-400` -> `text-muted-foreground`

**Borders:**
- `border-gray-*` -> `border-border` (or remove if redundant since border-border is the default)

**Status/semantic colors (keep the color but add dark variant):**
- `text-green-600` -> `text-green-600 dark:text-green-400`
- `text-red-600` -> `text-red-600 dark:text-red-400`
- `text-yellow-600` -> `text-yellow-600 dark:text-yellow-400`
- `bg-green-*` status badges -> add `dark:bg-green-900/30 dark:text-green-400` variants
- `bg-red-*` status badges -> add `dark:bg-red-900/30 dark:text-red-400` variants

**Hover states:**
- `hover:text-gray-900` -> `hover:text-foreground`

**Chart hex colors in Recharts components (team-overview.tsx and member-analytics.tsx):**
- `#3b82f6` (blue) -> `var(--chart-1)` (use hsl() or keep hex with dark: conditional -- but CSS variables are better). Since Recharts accepts CSS variable strings in stroke/fill, replace with `"var(--chart-1)"`.
- `#4ade80` (green for Low stress) -> `"var(--chart-2)"` or keep semantic green `#22c55e` / `#4ade80` as these are meaningful (stress = green/yellow/red). For stress colors, keep the hex values since they carry semantic meaning and are readable on both light and dark backgrounds.
- `#facc15` / `#eab308` (yellow for Medium stress) -> keep as-is (yellow is visible on both).
- `#f87171` / `#ef4444` (red for High stress) -> keep as-is.
- `#6b7280` (gray reference line) -> `"var(--chart-5)"` or `"var(--muted-foreground)"`.
- `#f97316` (orange trend line) -> `"var(--chart-3)"`.
