<objective>
Create two backend analytics endpoints that aggregate team report data over configurable time ranges (week/month/quarter). One endpoint returns team-level overview data (submission rates, hours, stress distribution, task volume by member). The other returns individual member analytics (streak, hours vs team avg, stress pattern, task output).

Purpose: Provide the data layer that the frontend analytics charts consume. All aggregation happens server-side to minimize client-side data processing.
Output: Two new API endpoints, shared analytics types, unit tests.
</objective>

```typescript
import type { StressLevel } from './task';

export type AnalyticsRange = 'week' | 'month' | 'quarter';

// --- Team Overview ---

export interface TeamAnalyticsSummary {
  submissionRate: number;        // 0-100 percentage
  submissionRateTrend: number;   // difference vs previous period (positive = up)
  avgHoursPerDay: number;
  avgHoursPerDayTrend: number;
  stressDistribution: { low: number; medium: number; high: number };
  taskCount: number;
  taskCountTrend: number;
}

export interface DailySubmissionRate {
  date: string;   // YYYY-MM-DD
  rate: number;    // 0-100
  submitted: number;
  total: number;
}

export interface HeatmapCell {
  userId: string;
  displayName: string;
  date: string;
  hours: number;
}

export interface DailyStressDistribution {
  date: string;
  low: number;
  medium: number;
  high: number;
}

export interface MemberTaskVolume {
  userId: string;
  displayName: string;
  taskCount: number;
}

export interface TeamAnalyticsResponse {
  range: AnalyticsRange;
  startDate: string;
  endDate: string;
  summary: TeamAnalyticsSummary;
  submissionRates: DailySubmissionRate[];
  heatmap: HeatmapCell[];
  stressTrend: DailyStressDistribution[];
  taskVolumeByMember: MemberTaskVolume[];
}

// --- Individual Member ---

export interface MemberAnalyticsSummary {
  submissionStreak: number;       // consecutive days submitted (counting back from today)
  avgHours: number;
  teamAvgHours: number;
  mostCommonStress: StressLevel | null;
  stressTrend: number;           // direction: -1 (improving), 0 (stable), 1 (worsening)
  avgTasksPerDay: number;
  teamAvgTasksPerDay: number;
}

export interface DailyHoursEntry {
  date: string;
  hours: number;
  teamAvg: number;
}

export interface DailyStressEntry {
  date: string;
  level: StressLevel | null;
}

export interface DailyTaskCount {
  date: string;
  count: number;
}

export interface SubmissionCalendarDay {
  date: string;
  submitted: boolean;
}

export interface MemberAnalyticsResponse {
  range: AnalyticsRange;
  startDate: string;
  endDate: string;
  userId: string;
  displayName: string;
  summary: MemberAnalyticsSummary;
  dailyHours: DailyHoursEntry[];
  stressTimeline: DailyStressEntry[];
  dailyTasks: DailyTaskCount[];
  submissionCalendar: SubmissionCalendarDay[];
}
```

2. Export all analytics types from `packages/shared/src/index.ts` — add a new section after the manager types.

3. Create `apps/api/src/manager/analytics.service.ts`:
   - Inject `SupabaseService`
   - Helper method `getDateRange(range: AnalyticsRange)` returns `{ startDate, endDate, prevStartDate, prevEndDate }`:
     - week: last 7 days ending today, previous 7 days before that
     - month: last 30 days ending today, previous 30 days before that
     - quarter: last 90 days ending today, previous 90 days before that
   - Method `getTeamAnalytics(teamId: string, range: AnalyticsRange): Promise<TeamAnalyticsResponse>`:
     - Query active team members (left_at IS NULL) from team_members
     - Query daily_reports for team in date range (and previous period for trends)
     - Query tasks for those reports
     - Aggregate:
       - submissionRate: (days with submission / (member count * days in range)) * 100
       - submissionRateTrend: current rate - previous period rate
       - avgHoursPerDay: sum of all task hours / days in range
       - stressDistribution: count of latest stress_level per member across submitted reports
       - taskCount: total tasks, trend = current - previous
       - submissionRates[]: per-day submission percentage
       - heatmap[]: each member x each day -> hours (from tasks)
       - stressTrend[]: per-day count of low/medium/high from submitted reports
       - taskVolumeByMember[]: count of tasks per member, sorted desc
   - Method `getMemberAnalytics(teamId: string, userId: string, range: AnalyticsRange): Promise<MemberAnalyticsResponse>`:
     - Query member's daily_reports + tasks in range
     - Query team averages for comparison
     - Calculate submission streak: count consecutive submitted days going backward from today
     - Calculate most common stress level (mode)
     - stressTrend: compare avg stress numeric value (low=1,med=2,high=3) in last half vs first half of period
     - dailyHours[]: member hours per day + team average per day
     - stressTimeline[]: stress level per day
     - dailyTasks[]: task count per day
     - submissionCalendar[]: each date in range -> submitted boolean
