export interface MemberAnalyticsSummary {

export interface DailyHoursEntry { date: string; hours: number; teamAvg: number; }

}

Create `apps/web/src/components/analytics/member-analytics.tsx`:

1. **Member selector:**
   - Fetch team members list using `useQuery` on `api.get<TeamReportsResponse>(`/teams/${teamId}/reports?date=${format(new Date(), 'yyyy-MM-dd')}`)` to get the list of members (reuse existing endpoint).
   - Render a `<select>` dropdown (styled with Tailwind: border, rounded, padding) listing all members by displayName (or email if no displayName).
   - State: `selectedUserId: string | null` -- default to first member when data loads.
   - When no member selected yet, show "Select a member to view analytics".

2. **Data fetching:**
   - When selectedUserId is set, `useQuery` with key `['member-analytics', teamId, selectedUserId, range]`, calls `api.get<MemberAnalyticsResponse>(`/teams/${teamId}/analytics/member/${selectedUserId}?range=${range}`)`
   - Show loading state while fetching

3. **Summary cards row** (same responsive grid as team overview: 1/2/4 cols):
   - Submission Streak: `{summary.submissionStreak} days` -- no trend arrow, just the number
   - Avg Hours: `{summary.avgHours}h` with subtitle `Team avg: {summary.teamAvgHours}h`. Color the value green if below team avg, red if above (indicating overwork)
   - Stress Pattern: Show `{mostCommonStress}` as text with the trend direction (arrow down = improving for stress, arrow up = worsening). Use SummaryCard with children slot to render a colored badge for the stress level
   - Task Output: `{summary.avgTasksPerDay}/day` with subtitle `Team avg: {summary.teamAvgTasksPerDay}/day`

4. **Charts** (2-column grid on lg, 1-column on mobile):

   a. **Hours Worked** (Recharts `ComposedChart`):
      - Bar for daily hours (`dailyHours[].hours`) in blue
      - Line for team average (`dailyHours[].teamAvg`) in orange, dashed
      - X-axis: dates (M/d), Y-axis: hours
      - Legend showing "Hours" and "Team Avg"
      - Tooltip with date, hours, team avg

   b. **Stress Timeline** (Recharts `LineChart` or `ScatterChart`):
      - Map stress levels to numeric: low=1, medium=2, high=3
      - Render as dots connected by line
      - Y-axis: custom tick labels (Low, Medium, High) at positions 1, 2, 3
      - Color dots: green for low, yellow for medium, red for high (use Recharts `dot` prop with custom render)
      - X-axis: dates
      - Skip null entries (days with no report)

   c. **Task Breakdown** (Recharts `BarChart`):
      - Vertical bars showing `dailyTasks[].count` per day
      - X-axis: dates, Y-axis: task count
      - Blue bars, tooltip with date and count

   d. **Submission Calendar** (GitHub-style grid, custom CSS implementation):
      - Render as a grid of small squares (14x14px or similar)
      - Layout: 7 rows (Mon-Sun), columns = weeks in the period
      - Green (`bg-green-500`) for submitted days, light gray (`bg-gray-200`) for missed days
      - Show day-of-week labels on the left (M, W, F)
      - Tooltip on hover showing the date and "Submitted" or "Missed"
      - Use CSS grid with `grid-template-rows: repeat(7, 1fr)` and auto columns
      - Data source: `submissionCalendar[]` -- map each date to its position based on day-of-week

All charts wrapped in Card with title. Handle empty state gracefully (no data message).
