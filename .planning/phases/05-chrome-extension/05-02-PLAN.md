<objective>
Build the Chrome extension as a new `apps/extension` workspace package with MV3 manifest, login flow, context menu task capture, and quick-add popup.

Purpose: This is the entire user-facing extension. Users right-click highlighted text on any page, select "Add to Daily Report", and the popup opens pre-filled with the text and URL. They enter hours and optional notes, then save.

Output: Working Chrome extension in `apps/extension/` that can be loaded unpacked in chrome://extensions for testing.
</objective>

<interfaces>
<!-- Backend endpoints this extension calls (from Plan 01) -->

POST /auth/extension-login
  Request: { email: string, password: string }
  Response: { accessToken: string, refreshToken: string }

POST /auth/extension-refresh
  Request: { refreshToken: string }
  Response: { accessToken: string, refreshToken: string }

GET /auth/me
  Headers: Authorization: Bearer {accessToken}
  Response: { userId: string, email: string }

GET /teams/my
  Headers: Authorization: Bearer {accessToken}
  Response: Array<{ id: string, name: string, role: string, ... }>

POST /tasks
  Headers: Authorization: Bearer {accessToken}
  Request: { title: string, estimatedHours: number, sourceLink?: string, notes?: string, reportDate: string, teamId: string }
  Response: { id: string, title: string, ... }

<!-- CreateTaskDto validation rules -->
title: string, minLength 1, maxLength 500
estimatedHours: number, min 0.01, max 24
sourceLink: optional URL, maxLength 2000
notes: optional string, maxLength 2000
reportDate: string, YYYY-MM-DD format
teamId: UUID
</interfaces>

<task type="auto">
  <name>Task 1: Scaffold extension workspace with build tooling and auth/API libraries</name>
  <files>apps/extension/package.json, apps/extension/tsconfig.json, apps/extension/vite.config.ts, apps/extension/manifest.json, apps/extension/src/lib/auth.ts, apps/extension/src/lib/api.ts, apps/extension/src/icons/icon-16.png, apps/extension/src/icons/icon-48.png, apps/extension/src/icons/icon-128.png</files>
  <action>
1. **Create workspace directory and package.json:**
   ```bash
   mkdir -p apps/extension/src/{popup,lib,icons}
   ```
   Create `apps/extension/package.json`:
   ```json
   {
     "name": "@daily-report/extension",
     "version": "1.0.0",
     "private": true,
     "type": "module",
     "scripts": {
       "dev": "vite build --watch",
       "build": "vite build"
     }
   }
   ```
   Run `pnpm install` from root, then:
   ```bash
   pnpm --filter @daily-report/extension add -D vite typescript @crxjs/vite-plugin@beta
   ```

2. **Create tsconfig.json** matching project conventions (ES2020 target, strict mode).

3. **Create vite.config.ts** using @crxjs/vite-plugin:
   ```typescript
   import { defineConfig } from 'vite';
   import { crx } from '@crxjs/vite-plugin';
   import manifest from './manifest.json';

   export default defineConfig({
     plugins: [crx({ manifest })],
     build: { outDir: 'dist' },
   });
   ```
   If CRXJS causes issues (beta plugin), fall back to plain Vite with manual manifest copy. The extension is simple enough to not depend on CRXJS.

4. **Create manifest.json** (MV3):
   ```json
   {
     "manifest_version": 3,
     "name": "Daily Report - Quick Add",
     "version": "1.0.0",
     "description": "Capture tasks from any webpage into your daily report",
     "minimum_chrome_version": "127",
     "permissions": ["contextMenus", "storage", "activeTab"],
     "host_permissions": ["http://localhost:3001/*"],
     "action": {
       "default_popup": "src/popup/popup.html",
       "default_icon": { "16": "src/icons/icon-16.png", "48": "src/icons/icon-48.png", "128": "src/icons/icon-128.png" }
     },
     "background": { "service_worker": "src/background.ts", "type": "module" },
     "icons": { "16": "src/icons/icon-16.png", "48": "src/icons/icon-48.png", "128": "src/icons/icon-128.png" }
   }
   ```

5. **Create placeholder icon PNGs** -- generate simple solid-color PNG files at 16x16, 48x48, 128x128 using a canvas approach or use simple placeholder SVG-to-PNG. A simple approach: create minimal valid PNG binary files (single-color squares). Use Node.js script or find placeholder approach that works without external deps. If creating real PNGs is impractical, use placeholder .png files (even 1-pixel PNGs renamed) and note they should be replaced with real icons.

6. **Create `apps/extension/src/lib/auth.ts`** -- token storage utilities using chrome.storage.local:
   - `saveTokens(accessToken, refreshToken)` -- stores both
   - `getAccessToken()` -- returns string | null
   - `getRefreshToken()` -- returns string | null
   - `clearTokens()` -- removes accessToken, refreshToken, userTeams, pendingTask
   - `isLoggedIn()` -- checks if accessToken exists
   - `saveUserTeams(teams)` -- stores team list
   - `getUserTeams()` -- retrieves team list

7. **Create `apps/extension/src/lib/api.ts`** -- fetch wrapper with Bearer auth:
   - `const API_URL = 'http://localhost:3001'`
   - `apiFetch<T>(endpoint, options)` -- attaches Authorization: Bearer header from stored token
   - On 401 response: call `tryRefresh()` which POSTs to `/auth/extension-refresh` with stored refresh token. If refresh succeeds, save new tokens and retry original request once. If refresh fails, clear tokens and throw 'Session expired'.
   - `extensionLogin(email, password)` -- calls POST /auth/extension-login, saves tokens, fetches GET /teams/my and saves teams.
   - `createTask(data)` -- calls POST /tasks with Bearer auth.

8. Verify build works: `cd apps/extension && pnpm build` (may have warnings about missing popup/background files -- that's fine, Task 2 adds those).

<task type="auto">
  <name>Task 2: Build background service worker and popup UI (login + quick-add)</name>
  <files>apps/extension/src/background.ts, apps/extension/src/popup/popup.html, apps/extension/src/popup/popup.ts, apps/extension/src/popup/popup.css</files>
  <action>
1. **Create `apps/extension/src/background.ts`** (service worker):
   - On `chrome.runtime.onInstalled`: create context menu item:
     ```typescript
     chrome.contextMenus.create({
       id: 'add-to-daily-report',
       title: 'Add to Daily Report',
       contexts: ['selection'],
     });
     ```
   - On `chrome.contextMenus.onClicked` for 'add-to-daily-report':
     - Store pending task data in chrome.storage.local: `{ pendingTask: { title: info.selectionText, sourceLink: info.pageUrl } }`
     - Open the popup programmatically: `chrome.action.openPopup()` (Chrome 127+ required per manifest minimum_chrome_version)
   - No in-memory state -- all state goes through chrome.storage.local (MV3 service worker termination safety).

2. **Create `apps/extension/src/popup/popup.html`**:
   - Minimal HTML shell: viewport meta, link to popup.css, `<div id="app">` container, script tag for popup.ts (type="module").
   - Inside #app, two sections (shown/hidden via JS):
     - `#login-view`: email input, password input, login button, error message area, app title/branding
     - `#task-view`: header showing user email + logout button, form with: title (textarea, pre-filled), sourceLink (input, pre-filled, editable), estimatedHours (number input, step="0.25", min="0.25", max="24"), notes (textarea, optional), save button, status/error area
   - Set popup width/height via CSS (min-width: 360px, min-height: 400px for comfortable form).

3. **Create `apps/extension/src/popup/popup.ts`**:
   - On DOMContentLoaded:
     - Check `isLoggedIn()` from auth.ts
     - If logged in: show #task-view, hide #login-view, load pending task from storage, pre-fill form
     - If not logged in: show #login-view, hide #task-view

   - **Login flow:**
     - On login form submit: call `extensionLogin(email, password)` from api.ts
     - On success: switch to task view, load pending task if any
     - On error: show error message

   - **Quick-add flow:**
     - On popup open: read `pendingTask` from chrome.storage.local. If exists, pre-fill title and sourceLink fields, then clear pendingTask from storage.
     - If no pendingTask, show empty form (user clicked icon directly).
     - reportDate: auto-set to today's date in YYYY-MM-DD format.
     - teamId: auto-select first team from stored userTeams (matching web app's auto-select-first-team behavior per Phase 3 decision). If user has multiple teams, show a team dropdown select.
     - On save: validate form (title required, estimatedHours required and > 0), call `createTask({ title, estimatedHours, sourceLink, notes, reportDate, teamId })`.
     - On success: show success message ("Task added!"), clear form after 1.5 seconds, optionally close popup.
     - On error: show error message. If 'Session expired', switch to login view.

   - **Logout:**
     - On logout button click: call `clearTokens()`, switch to login view.

4. **Create `apps/extension/src/popup/popup.css`**:
   - Clean, minimal styling. Body background white, system font stack.
   - Form inputs: full width, padding 8px, border 1px solid #ddd, border-radius 4px.
   - Primary button: blue background (#2563eb), white text, full width, padding 10px, border-radius 4px.
   - Error messages: red text. Success messages: green text.
   - Header area with user email and logout link (small, right-aligned).
   - Responsive within popup constraints (360px width).

5. **Build and verify:** Run `cd apps/extension && pnpm build`. The dist/ folder should contain the built extension. Verify manifest.json is in dist/.
