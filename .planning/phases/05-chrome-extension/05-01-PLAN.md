<objective>
Add Bearer token authentication support to the NestJS API so the Chrome extension can authenticate without cookies.

Purpose: The web app uses httpOnly cookies for JWT auth, but Chrome extensions cannot send httpOnly cookies cross-origin. The extension needs Bearer token auth via Authorization header, plus dedicated login/refresh endpoints that return tokens in the response body.

Output: Modified Passport strategies with dual extraction, new extension-login and extension-refresh endpoints, updated CORS config, and tests.
</objective>

     app.enableCors({
       origin: (origin, callback) => {
         const allowedOrigins = [process.env.FRONTEND_URL];
         // Allow requests with no origin (Chrome extension, Postman, server-to-server)
         if (!origin || allowedOrigins.includes(origin) || origin.startsWith('chrome-extension://')) {

           callback(new Error('Not allowed by CORS'));

       },
       credentials: true,
