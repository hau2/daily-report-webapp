    - GET /users/me: returns user profile (id, email, displayName, timezone, emailVerified, createdAt) from database. Excludes password_hash and refresh_token_hash.
    - PATCH /users/me with { displayName }: updates display name, returns updated profile
    - PATCH /users/me with { email, currentPassword }: validates current password, updates email, returns updated profile
    - PATCH /users/me with { email } without currentPassword: returns 400 "Current password required to change email"
    - PATCH /users/me with { newPassword, currentPassword }: validates current password, hashes new password, updates password_hash, returns updated profile
    - PATCH /users/me with { newPassword } without currentPassword: returns 400 "Current password required to change password"
    - PATCH /users/me with wrong currentPassword: returns 401 "Invalid current password"
  </behavior>
  <action>
    1. Add shared schemas and types for profile update:
       - In `packages/shared/src/schemas/auth.schema.ts`, add:
         ```typescript
         export const updateProfileSchema = z.object({
           displayName: z.string().min(1).max(100).optional(),
           email: z.string().email().optional(),
           newPassword: z.string().min(8).optional(),
           currentPassword: z.string().min(1).optional(),
           timezone: z.string().optional(),
         });
         ```
       - In `packages/shared/src/types/auth.ts`, add:
         ```typescript
         export interface UpdateProfileRequest {
           displayName?: string;
           email?: string;
           newPassword?: string;
           currentPassword?: string;
           timezone?: string;
         }
         ```
       - Update `packages/shared/src/index.ts` to export updateProfileSchema, UpdateProfileRequest, UpdateProfileInput

    2. Create `apps/api/src/users/dto/update-profile.dto.ts`:
       - Class with optional fields: displayName (IsString, IsOptional), email (IsEmail, IsOptional), newPassword (IsString, MinLength(8), IsOptional), currentPassword (IsString, IsOptional), timezone (IsString, IsOptional)

    3. Create `apps/api/src/users/users.service.ts`:
       - Inject SupabaseService
       - `getProfile(userId: string)`:
         a. Query supabase.from('users').select('id, email, display_name, timezone, email_verified, created_at, updated_at').eq('id', userId).single()
         b. Map database column names to camelCase (display_name -> displayName, email_verified -> emailVerified, etc.)
         c. Return mapped User object (exclude password_hash and refresh_token_hash)
       - `updateProfile(userId: string, dto: UpdateProfileDto)`:
         a. If dto.email or dto.newPassword is provided, currentPassword is REQUIRED -- throw BadRequestException if missing
         b. If currentPassword provided: fetch user including password_hash, verify with argon2.verify. If wrong, throw UnauthorizedException('Invalid current password')
         c. Build update object:
            - If dto.displayName: set display_name
            - If dto.email: set email
            - If dto.newPassword: hash with argon2, set password_hash, also nullify refresh_token_hash (force re-login)
            - If dto.timezone: set timezone
         d. Execute supabase.from('users').update(updates).eq('id', userId).select().single()
         e. Map and return updated User object

    4. Create `apps/api/src/users/users.controller.ts`:
       - @Controller('users')
       - @UseGuards(AccessTokenGuard) on the controller class (all endpoints protected)
       - GET /users/me -- extract userId from req.user, call usersService.getProfile(userId)
       - PATCH /users/me -- @Body() UpdateProfileDto, extract userId from req.user, call usersService.updateProfile(userId, dto)

    5. Create `apps/api/src/users/users.module.ts`:
       - @Module with UsersController and UsersService

    6. Update `apps/api/src/app.module.ts`:

       - Test: 'update email - requires current password' -- call without currentPassword, expect BadRequestException
       - Test: 'update email - validates current password' -- provide wrong password, mock argon2.verify false, expect UnauthorizedException
       - Test: 'update email - succeeds with correct password' -- mock argon2.verify true, verify supabase update with new email
       - Test: 'update password - requires current password and hashes new password' -- verify argon2.hash called with newPassword, supabase update called with new hash
       - Test: 'update password - invalidates refresh token' -- verify refresh_token_hash set to null
