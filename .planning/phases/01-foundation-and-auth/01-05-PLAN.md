  <behavior>
    - updateProfile with displayName: Updates display name, returns updated user (no password required)
    - updateProfile with timezone: Updates timezone, returns updated user (no password required)
    - updateProfile with email: Requires currentPassword, verifies it with Argon2, updates email
    - updateProfile with email but wrong currentPassword: Throws UnauthorizedException
    - updateProfile with email but missing currentPassword: Throws BadRequestException
    - updateProfile with newPassword: Requires currentPassword, verifies it, hashes new password with Argon2, updates
    - updateProfile with newPassword but wrong currentPassword: Throws UnauthorizedException
    - updateProfile with duplicate email: Throws ConflictException (409)
  </behavior>
  <action>
    **Step 1: Create UpdateProfileDto** at `apps/api/src/users/dto/update-profile.dto.ts`:
    - All fields optional: `@IsOptional() @IsString() displayName`, `@IsOptional() @IsEmail() email`, `@IsOptional() @IsString() currentPassword`, `@IsOptional() @MinLength(8) newPassword`, `@IsOptional() @IsString() timezone`.
    - Add validation: if `email` or `newPassword` is provided, `currentPassword` must also be provided. Use a custom validator or handle in the service.

    **Step 2: Extend UsersService** with `updateProfile(userId: string, dto: UpdateProfileDto)`:
    - Find user by id (throw NotFoundException if not found).
    - Build update data object:
      - If `displayName` provided, add to update.
      - If `timezone` provided, add to update.
      - If `email` provided: require `currentPassword` (throw BadRequestException if missing), verify current password with argon2.verify against user.passwordHash (throw UnauthorizedException if wrong), check email uniqueness (throw ConflictException if taken), add to update.
      - If `newPassword` provided: require `currentPassword` (throw BadRequestException if missing), verify current password (throw UnauthorizedException if wrong), hash new password with argon2, add passwordHash to update.
    - Execute prisma.user.update with the built data.
    - Return updated user (exclude passwordHash and refreshTokenHash from response).

    **Step 3: Create UsersController** at `apps/api/src/users/users.controller.ts`:
    - `@Controller('users')`.
    - `PATCH /profile`: Use `@UseGuards(JwtAuthGuard)`. Use `@CurrentUser() user`. Call usersService.updateProfile(user.id, dto). Return updated profile.
    - `GET /profile`: Use `@UseGuards(JwtAuthGuard)`. Use `@CurrentUser() user`. Return full profile from usersService.findById (exclude sensitive fields).

    **Step 4: Update UsersModule:**
    - Add UsersController.
    - Import necessary modules.

    **Step 5: Update AppModule** to import UsersModule (if not already done -- it may already be imported via AuthModule dependency).

    **Step 6: Fill in users.service.spec.ts tests** (replace todo stubs):
    - "update name - changes display name": provide displayName only, verify prisma.user.update called with displayName.
    - "update email - requires current password": provide email without currentPassword -> BadRequestException. Provide email with correct currentPassword -> updates email.
    - "update email - rejects wrong current password": provide email with wrong currentPassword -> UnauthorizedException.
    - "update email - rejects duplicate email": provide email that already exists -> ConflictException.
    - "update password - requires current password and hashes new password": provide newPassword with correct currentPassword -> verify argon2.hash called, prisma.user.update called with new passwordHash.
    - "update password - rejects wrong current password": provide newPassword with wrong currentPassword -> UnauthorizedException.
    - "update timezone - changes timezone": provide timezone only, verify update.
    - Mock PrismaService and argon2.
