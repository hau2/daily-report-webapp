  <behavior>
    - EmailService.sendVerificationEmail: calls Resend with verification URL containing JWT token
    - EmailService.sendPasswordResetEmail: calls Resend with reset URL containing JWT token
    - register: after creating user, generates verification JWT (1h expiry, purpose: 'email-verification', sub: userId) and calls emailService.sendVerificationEmail
    - verify-email: decodes JWT, checks purpose claim, updates email_verified=true in database
    - verify-email with expired/invalid token: returns 400 BadRequestException
    - forgot-password: finds user by email, generates reset JWT (1h expiry, purpose: 'password-reset', sub: userId), sends email. Returns 200 even if email not found (prevent enumeration)
    - reset-password: decodes JWT, checks purpose claim, hashes new password, updates password_hash in database
    - reset-password with expired/invalid token: returns 400 BadRequestException
  </behavior>
  <action>

       - sendPasswordResetEmail(to: string, token: string): sends email with link `${FRONTEND_URL}/reset-password?token=${token}`
       - HTML templates: simple, clean HTML with the link. No complex templates needed for v1.
       - If RESEND_API_KEY is not set (development without Resend), log the verification/reset URL to console instead of sending email. This allows development without Resend configured.

       - `reset-password.dto.ts`: { token: string, password: string } (IsString + MinLength(8))

    4. Update `apps/api/src/auth/auth.service.ts` -- add 3 methods:
       - Inject EmailService (add to constructor)
       - Update `register()`: after successful user creation, generate verification JWT with { sub: user.id, purpose: 'email-verification' } expiring in '1h', call emailService.sendVerificationEmail(user.email, token)
       - Add `verifyEmail(dto: VerifyEmailDto)`:
         a. Decode token with jwtService.verifyAsync(dto.token, { secret: JWT_SECRET })
         b. Check payload.purpose === 'email-verification', else throw BadRequestException
         c. Update user: set email_verified = true where id = payload.sub
         d. Return { message: 'Email verified successfully' }
       - Add `forgotPassword(dto: ForgotPasswordDto)`:
         a. Find user by email. If not found, still return success (prevent email enumeration)
         b. If found: generate reset JWT with { sub: user.id, purpose: 'password-reset' } expiring in '1h'
         c. Call emailService.sendPasswordResetEmail(user.email, token)
         d. Return { message: 'If an account exists, a reset link has been sent' }
       - Add `resetPassword(dto: ResetPasswordDto)`:
         a. Decode token with jwtService.verifyAsync(dto.token, { secret: JWT_SECRET })
         b. Check payload.purpose === 'password-reset', else throw BadRequestException
         c. Hash new password with argon2.hash(dto.password)
         d. Update user: set password_hash where id = payload.sub
         e. Also nullify refresh_token_hash (force re-login after password change)
         f. Return { message: 'Password reset successfully' }

    5. Update `apps/api/src/auth/auth.controller.ts` -- add 3 endpoints:

       - POST /auth/reset-password -- @Body() ResetPasswordDto, calls authService.resetPassword
