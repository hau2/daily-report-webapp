    - sendVerificationEmail: Creates JWT token (type: email-verification, 24h expiry, JWT_VERIFICATION_SECRET), calls mailService
    - verifyEmail: Validates JWT token, marks user emailVerified=true, throws on invalid/expired token
    - verifyEmail: Throws on wrong token type (not email-verification)
    - sendPasswordResetEmail: Creates JWT token (type: password-reset, 1h expiry, JWT_RESET_SECRET), calls mailService
    - sendPasswordResetEmail: Does NOT throw when email not found (security: don't reveal if email exists)
    - resetPassword: Validates JWT token, hashes new password with Argon2, updates user
    - resetPassword: Throws on expired/invalid token
    - resendVerification: Finds user by email, sends new verification email if not already verified
    - login: Returns specific error when user exists but emailVerified=false
  </behavior>

    - `reset-password.dto.ts`: `@IsString()` token, `@MinLength(8)` password.

    **Step 5: Extend AuthService** with new methods:
    - `sendVerificationEmail(user)`: Generate JWT with `{ sub: user.id, type: 'email-verification' }`, sign with `JWT_VERIFICATION_SECRET`, expiry 24h. Build URL: `FRONTEND_URL/verify-email?token=TOKEN`. Call mailService.sendVerificationEmail().
    - `verifyEmail(token: string)`: Verify JWT with JWT_VERIFICATION_SECRET. Check payload.type === 'email-verification'. Update user emailVerified=true. Throw UnauthorizedException on invalid token.
    - `sendPasswordResetEmail(email: string)`: Find user by email. If not found, return silently (security). Generate JWT with `{ sub: user.id, type: 'password-reset' }`, sign with `JWT_RESET_SECRET`, expiry 1h. Build URL: `FRONTEND_URL/reset-password?token=TOKEN`. Call mailService.sendPasswordResetEmail().
    - `resetPassword(token: string, newPassword: string)`: Verify JWT with JWT_RESET_SECRET. Check payload.type === 'password-reset'. Hash new password with Argon2. Update user passwordHash. Throw UnauthorizedException on invalid token.
    - `resendVerification(email: string)`: Find user. If not found or already verified, return silently. Otherwise send verification email.

    **Step 6: Modify `register()` method** in AuthService:
    - After creating the user, call `this.sendVerificationEmail(user)`.

    **Step 7: Modify `validateUser()` method** in AuthService:

    - "forgot - returns silently for unknown email"
    - "reset - validates token and updates password"
    - "reset - rejects expired token"
    - Mock MailService in all tests (never actually send email).

    **Step 9: Update AuthModule** to import MailModule.
