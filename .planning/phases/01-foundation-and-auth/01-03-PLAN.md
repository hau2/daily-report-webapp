<objective>
Build the Next.js BFF proxy layer and all authentication frontend pages (login, register) with react-hook-form validation, plus a protected dashboard layout with auth guards.

Purpose: This connects the browser to the NestJS auth backend through same-origin proxy routes, ensuring httpOnly cookies work seamlessly. The browser never talks to NestJS directly, preventing the auth token desync pitfall from the research.

Output: Working login and register pages, BFF proxy, protected dashboard shell, and session persistence.
</objective>

<!-- From Plan 01-02: NestJS auth endpoints the BFF proxy must forward to -->
POST /auth/register  - Body: { email, password, displayName? } -> 201 { id, email, displayName, emailVerified }
POST /auth/login     - Body: { email, password } -> 200 { message } + Set-Cookie (access_token, refresh_token)
POST /auth/refresh   - Cookie: refresh_token -> 200 { message } + Set-Cookie (access_token)
POST /auth/logout    - Cookie: access_token -> 200 { message } + Clear-Cookie
GET  /auth/me        - Cookie: access_token -> 200 { id, email, displayName, emailVerified, timezone }

  <action>
    **Step 1: Create BFF proxy for auth routes** at `apps/web/src/app/api/auth/[...path]/route.ts`:
    - Implement `proxyRequest(request: NextRequest, method: string)` function per the research Pattern 2.
    - Read `NESTJS_API_URL` from `process.env`.
    - Strip `/api` prefix from pathname to forward to NestJS (e.g., `/api/auth/login` -> `NESTJS_URL/auth/login`).
    - Forward the `cookie` header from the incoming request to NestJS.
    - Forward the `content-type` header.
    - Forward request body for non-GET methods.
    - Pipe back ALL response headers from NestJS, especially `set-cookie` headers (critical for httpOnly cookie propagation).
    - Export GET, POST, PATCH, DELETE handlers that call proxyRequest.

    **Step 2: Create BFF proxy for users routes** at `apps/web/src/app/api/users/[...path]/route.ts`:
    - Same proxy pattern as auth routes but for `/api/users/*` -> `NESTJS_URL/users/*`.

    **Step 3: Create client-side API wrapper** at `apps/web/src/lib/api.ts`:
    - Export an `api` object with methods: `get(path)`, `post(path, body)`, `patch(path, body)`, `delete(path)`.
    - Base URL is `/api` (relative, goes through BFF proxy).
    - All requests include `credentials: 'include'` (for cookies) and `Content-Type: application/json`.
    - Parse JSON response. On non-ok responses, throw an error with the response body message.

    **Step 4: Create auth utility** at `apps/web/src/lib/auth.ts`:
    - `getSession()`: Server-side function that calls `NESTJS_API_URL/auth/me` with forwarded cookies from `headers()` (Next.js server function). Returns user object or null.
    - This function is used by Server Components and layouts to check auth state during SSR.
    - Import `cookies` from `next/headers` to forward the cookie header.

    **Step 5: Update `apps/web/next.config.ts`:**
    - Add `NESTJS_API_URL` to `env` config if needed (or rely on .env.local).
    - No special proxy config needed since we use Route Handlers.
