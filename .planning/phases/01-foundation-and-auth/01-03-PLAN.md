<!-- From Plan 02: Auth API endpoints -->
Auth API contract (NestJS backend on port 3001):
- POST /auth/register { email, password, displayName? } -> 201 { message }
- POST /auth/login { email, password } -> 200 { message } + sets httpOnly cookies
- POST /auth/refresh -> 200 { message } + refreshes cookies (requires refresh_token cookie)
- POST /auth/logout -> 200 { message } + clears cookies (requires access_token cookie)
- GET /users/me -> 200 { id, email, displayName, timezone, emailVerified } (requires access_token cookie)
  Note: /users/me endpoint will be created in Plan 05, but the dashboard can attempt to call it
  and handle 404 gracefully until then. For now, use a simpler auth check approach.
