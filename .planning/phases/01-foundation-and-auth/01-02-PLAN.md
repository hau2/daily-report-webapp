  <behavior>
    - POST /auth/register: Accepts {email, password, displayName?}, returns 201 with user object (no passwordHash)
    - POST /auth/login: Accepts {email, password}, sets httpOnly access_token cookie (15min, path: /) and refresh_token cookie (30d, path: /auth/refresh), returns { message: 'Login successful' }
    - POST /auth/refresh: Reads refresh_token from cookie, sets new access_token cookie, returns { message: 'Token refreshed' }
    - POST /auth/logout: Clears both cookies, clears DB refresh hash, returns { message: 'Logged out' }
    - GET /auth/me: Returns current user from JWT (requires valid access_token cookie)
    - Cookie config: httpOnly: true, secure: process.env.NODE_ENV === 'production', sameSite: development ? 'lax' : 'strict'
  </behavior>
  <action>
    **Step 1: Create Passport strategies:**

    `jwt.strategy.ts` (for access token):
    - Extract JWT from httpOnly cookie `access_token` first, fallback to Authorization Bearer header (for future Chrome extension).
    - Secret from `JWT_ACCESS_SECRET` config.
    - validate() returns `{ id: payload.sub, email: payload.email }`.

    `jwt-refresh.strategy.ts` (for refresh token):
    - Extract from httpOnly cookie `refresh_token` only.
    - Secret from `JWT_REFRESH_SECRET` config.
    - validate() attaches `refreshToken` (raw token from cookie) to the request for hash comparison. Use `passReqToCallback: true` to access the request.

    `local.strategy.ts` (for login):
    - Call `authService.validateUser(email, password)`.
    - Return user or throw UnauthorizedException.
    - Set `usernameField: 'email'` in strategy options.

    **Step 2: Create guards:**
    - `jwt-auth.guard.ts`: extends AuthGuard('jwt').
    - `jwt-refresh.guard.ts`: extends AuthGuard('jwt-refresh').
    - `local-auth.guard.ts`: extends AuthGuard('local').

    **Step 3: Create AuthController** at `apps/api/src/auth/auth.controller.ts`:
    - `@Controller('auth')`.
    - `POST /register`: Call authService.register(dto), return 201.
    - `POST /login`: Use @UseGuards(LocalAuthGuard). Call authService.generateTokens(user). Set cookies on response using `res.cookie()`. Return success message. Cookie config must differ by environment: in development use `secure: false`, `sameSite: 'lax'`; in production use `secure: true`, `sameSite: 'strict'`. Access token path: `/`. Refresh token path: `/auth/refresh`.
    - `POST /refresh`: Use @UseGuards(JwtRefreshAuthGuard). Get userId and refreshToken from request.user. Call authService.refreshTokens(). Set new cookies. Return success.
    - `POST /logout`: Use @UseGuards(JwtAuthGuard). Call authService.logout(userId). Clear both cookies with `res.clearCookie()`. Return success.
    - `GET /me`: Use @UseGuards(JwtAuthGuard). Use @CurrentUser() decorator. Return user from usersService.findById (exclude passwordHash, refreshTokenHash).
